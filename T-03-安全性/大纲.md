# Claude Code 源码解析 - 安全性

## 文章大纲

### 引言
- AI 驾驭工程中的安全问题
- Claude Code 的多层安全防护体系

### 一、权限模式：用户体验与安全的平衡

> 来源：`src/types/permissions.ts`, `src/utils/permissions/PermissionMode.ts`

#### 1.1 五种权限模式
- **default** — 标准模式，每次危险操作询问
- **acceptEdits** — 自动接受文件编辑
- **plan** — 计划模式，只读不执行
- **bypassPermissions** — 绕过所有权限检查
- **dontAsk** — 不要询问

#### 1.2 自动模式（Ant 专有）
- feature flag: `TRANSCRIPT_CLASSIFIER`
- AI 辅助判断是否需要询问
- 非阻塞式安全评估

### 二、权限规则系统：精细化控制

> 来源：`src/types/permissions.ts`

#### 2.1 规则结构
```typescript
type PermissionRule = {
  source: 'userSettings' | 'projectSettings' | 'cliArg' | 'command'
  ruleBehavior: 'allow' | 'deny' | 'ask'
  ruleValue: { toolName: string; ruleContent?: string }
}
```

#### 2.2 规则来源优先级
1. `cliArg` — 命令行参数
2. `command` — 会话中的命令
3. `userSettings` — 用户配置
4. `projectSettings` — 项目配置（需信任）

#### 2.3 规则匹配
- 精确匹配工具名
- 支持 glob 模式
- deny 优先于 allow

### 三、危险模式检测：保护敏感操作

- `kubectl`, `aws`, `gcloud` — 云资源操作

### 四、路径验证：文件系统安全

> 来源：`src/utils/permissions/pathValidation.ts`

#### 4.1 路径验证流程
1. 检查 deny 规则
2. 检查内部可编辑路径
3. 安全检查（Windows 模式、危险文件）
4. 工作目录检查
5. 检查 allow 规则

#### 4.2 危险路径检测
- `rm *` — 通配符删除
- `/` — 根目录
- `~` — 家目录
- Windows 驱动器根目录

#### 4.3 路径遍历防护
- 拒绝 `..` 遍历
- 拒绝 UNC 网络路径
- 拒绝 shell 扩展语法（`$VAR`, `%VAR%`）

### 五、安全决策原因

```typescript
type PermissionDecisionReason =

  | { type: 'subcommandResults' }
  | { type: 'classifier'; classifier: string }

```

### 六、设计思想

#### 6.1 纵深防御
- 多层检查，不依赖单一安全机制
- deny 规则优先
- 自动模式辅助决策

#### 6.2 用户体验平衡
- 权限模式让用户选择信任级别
- 常见操作可预批准
- AI 辅助减少不必要的询问

#### 6.3 透明可控
- 每个决策都有原因
- 用户可查看和修改规则
- 安全检查可审计
