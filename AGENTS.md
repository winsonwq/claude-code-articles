# AGENTS.md - Claude Code 源码解析系列

## 使命

通过代码解析，深入理解 Claude Code 的 AI 驾驭工程思想，输出高质量技术文章。

## 文章写作规则

### 1. 调研深度
- 每天文章是深度调研编写
- 源代码中的角角落落都需要仔细查看
- 不遗漏任何细节

### 2. 代码即证据
- 使用代码来诠释思想，有依据
- 每一个观点都要有对应的源码引用
- 引用格式：`src/path/to/file.ts:行号`

### 3. 概念注释
- 遇到模糊的概念或专业术语
- 需要网络搜索其意思作为注释
- 尤其关注 harness engineering（AI 驾驭工程）领域的相关思想和概念
- 注释格式：`> [概念名] - 来源/解释`

### 4. 写作流程

```
调研收集 → 编写大纲 → 人为确认 → 深度续写 → review → 按 wechat-publish 技能检查 → 发布到公众号 → 检查后优化
```

**各阶段说明：**

| 阶段 | 说明 |
|------|------|
| 调研收集 | 深入阅读源码，收集相关代码片段、架构设计 |
| 编写大纲 | 整理文章结构，关键点列表 |
| 人为确认 | 向阿秋确认大纲方向是否正确 |
| 深度续写 | 按大纲逐节深度编写 |
| review | 自我审查代码引用的准确性 |
| wechat-publish 检查 | 使用 wechat-publish 技能检查格式 |
| 发布到公众号 | 推送到公众号草稿箱 |
| 检查后优化 | 发布后根据效果优化 |

### 5. 写作规范

- 文章标题格式：`【Claude Code 解析】XXX 的设计与实现`
- 每个技术点都要有代码示例
- 复杂的架构图可以用文字描述
- 结尾要有思考延伸
